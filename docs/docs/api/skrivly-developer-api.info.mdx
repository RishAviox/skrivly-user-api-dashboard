The Skrivly Developer API lets you programmatically interact with your organization's documents, signers, and workflows using a Developer API Key.

## Authentication
All endpoints require a Bearer token (your Developer API Key):
```
Authorization: Bearer YOUR_API_KEY
```
Get your API key from the Developer Dashboard → **Developer API Keys** section.

## Workflow Overview

Sending a document for signing requires three steps:

1. **Create Document** – Upload the PDF
2. **Add Signers** – Define who needs to sign
3. **Send for Signing** – Trigger email notifications to signers
